# AUDIT · 潮汐图书馆现状弱点清单（2026-06-30）

> 方法：用 **Playwright 真实浏览器**（1440×900，非隐藏预览页）加载 dev server，经 DevBridge
> `__freecam` 把相机摆到 7 个机位实拍（入场页 / 建立镜头 / 低头水面 / 书墙近景 / 留声机近景 /
> 浮岛物件 / 木梯）。证据图在 `references/_audit/`。按"严重度"排序，最严重在前；不打分、不下"通过"。
> 维度参照六维 3D 标准：①光照与氛围 ②材质与密度 ③相机手感 ④声音 ⑤整体与灵魂 ⑥性能。

---

## S0 · 体验级硬伤（用户已二次/三次点名，先修）

### A1　点击物件聚焦"不以物件为中心" + 退出无转场（硬切）　[③相机手感]
- 现象：点一个物件，镜头不是绕该物件居中，而是飞到**整个功能区的锚点**；退出时画面**瞬切**回漫游，无过渡。
- 根因：`PlayerControls` 聚焦分支只读 `ZONE_ANCHORS[zoneId]`（区级锚点），与"点中的具体物件"无关；退出转场虽已写（0.55s 缓动）但**用户仍报硬切**——需真机逐帧验证它到底有没有生效，没生效就是 bug。
- 证据：代码 `PlayerControls.tsx` 聚焦分支；`store.focusZone(id)` 只传 zoneId。

### A2　空气墙"弹回来"、碰撞不滑动　[③相机手感]
- 现象：走到边界/浮岛旁，不是"被挡住贴着走"，而是走进去一半再被**弹回**；撞墙不顺。
- 根因：`walk.ts` 用**径向夹回**（court 圆周）+ **径向弹出**（浮岛圆）——都不沿墙**滑动**(slide)；斜向撞击会被推回，毫无"贴墙滑行"手感。
- 证据：`walk.ts` `clampToWalkable` / `pushOut`。

## S1 · 视觉廉价感的核心（"还是丑丑的"）

### B1　灯具是扁平 2D 圆盘"棒棒糖"——全场最廉价的一处　[②材质与密度]
- 现象：每盏灯都是**一个正对相机的扁平亮圆盘 + 一根杆**，毫无体积，像贴片。出现在每一张截图里。
- 证据：`audit_hero` / `audit_bookwall`（书墙前那盏）/ `audit_pedestals` / `audit_gramo`（左侧）。
- 方向：换成**有体积的 3D 灯**（灯罩几何 + 内壁自发光 + 暖点光 + bloom），或引入手工低多边形台灯/灯笼模型。这是 ROI 最高的单点。

### B2　一切"漂浮、互不相连"，没有接地与实体感　[⑤整体与灵魂 / 构图]
- 现象：浮岛是**漂浮的黑色板**；书**悬空**在水上；书墙是**带缝的浮动隔板**，缝里直接透出星空——"墙"不成墙。整个场景像零件散落在太空。
- 证据：`audit_hero`（散落感）/ `audit_water`（书、灯漂在星空里）/ `audit_bookwall`（隔板间透星空）/ `audit_pedestals`（漂浮黑板）。
- 方向：给元素**接地阴影 / 落地结构 / 体量**；书墙加**背板**、书**坐实**在隔板；浮岛给**实体基座+水面接触**（涟漪/倒影/雾脚），让"漂浮"成为有意为之的构图而非散乱。

### B3　留声机喇叭内壁是"黑洞"、金属又平又黑、还有过曝白斑　[①光照 / ②材质]
- 现象：近看喇叭口**内壁全黑**（看进去是黑洞）；黄铜金属**反射不到东西、发死黑**；唱盘/唱针处又有一块**刺眼白斑**。
- 根因：模型喇叭材质**单面**（背面剔除）+ 喉部点光照不到内壁；金属 `metalness≈0.9` 但 IBL 只是 3 个 Lightformer 的**薄环境**，没东西可反射 → 死黑；点光太集中 → 金属过曝。
- 证据：`audit_gramo`。
- 方向：喇叭材质 `DoubleSide` + 喉光重打；**换更厚的 Environment（CC0 夜景 HDRI 或更强的程序化 env）**让金属有反射；点光调柔、提 roughness。

### B4　整体太暗，暖光是孤立水洼，缺统一氛围光/体积光　[①光照与氛围]
- 现象：大片区域近黑，几处暖光彼此不连，没有"一个空间被同一种光统一笼罩"的氛围；无体积光/雾光。
- 证据：全部截图（尤其 `audit_stairs` 台阶发黑、`audit_pedestals` 大片死黑）。
- 方向：抬环境光/补光的"地板"，加**体积光/雾中光锥**与统一色温分级，让暗部有信息、暖冷对比成体系。

## S2 · 中等（影响质感与舒适）

### C1　镜面水太透明，没有"水面存在感"——像漂在太空　[②材质]
- 现象：低头几乎看不到"水"，只看到镜像星空；缺少水面的法线扰动/掠射反光/边缘，"走在水上"更像"飘在星空里"。
- 证据：`audit_water`。
- 方向：给水面加**极淡的法线扰动 + 掠射菲涅尔反光 + 一点雾脚**，既保留星海镜面又有水的存在。

### C2　低头边走时涟漪"抽搐"　[⑥性能 / ③相机]
- 现象：用户报"一直低头边走边看水波时，水波会一直抽搐"，疑似帧率/时序。
- 方向：查 `ripples`/`Water` 涟漪 shader 的时间项与 dpr/帧率耦合；用稳定时钟、避免与 headbob/相机抖动叠加。

### C3　台阶"体感"与台阶模型未配准　[③相机手感 / 碰撞]
- 现象：能下台阶（✓），但一级一级的体感与可见台阶**没对齐**，视角反馈弱。
- 根因：`walk.ts` `stepHeightAt` 的离散台阶高度与视觉台阶几何/相机平滑未严格对应。
- 证据：`audit_stairs`。

### C4　书架无背板、书未坐实（部分悬浮）　[②材质]
- 现象：隔板是浮板、无背，书有的浮在板上方。
- 证据：`audit_bookwall`。
- 方向：加背板、把书 Y 对齐到隔板面。（与 B2 同源。）

## S3 · 低 / 可延后（含用户明确"可等"）

- **D1 望远镜是静态摆设**（观星台右侧）——用户说"可以更有创意，但可再等等"。延后，但留作"点击有回报"的候选。〔④/⑤〕
- **D2 音乐单曲**——用户说"挺好听，能不能多加一点"。加 1–2 首 CC0/PD 曲，做切换。〔④声音〕
- **D3 入场遮罩不透明**，遮住背后正在环绕的世界，弱化"进门头 60 秒"情绪弧。可让遮罩半透/露出环绕镜头。〔⑤〕

## 修复优先级（本轮）
1. B3 留声机喇叭/金属/Environment（高 ROI，已点名）
2. B1 扁平灯 → 3D 灯具（全场最廉价）
3. B2/C4 接地与书墙背板 + 坐实书（除散乱感）
4. A2 碰撞滑动（除"弹回来"）
5. A1 聚焦以物件为中心 + 退出转场真机验证/修
6. B4 统一氛围光提亮 + C1 水面存在感
7. C2 涟漪抽搐、C3 台阶配准（查清再修）
8. 延后：D1 望远镜互动、D2 多曲、D3 入场遮罩
